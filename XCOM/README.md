# XCOM 


XCOM supports seamless, high-speed, and secure data transfer across a wide variety of systems (mainframe, Windows, Linux, AIX, AS/400, cloud storage, etc.). It uses advanced data compression and record packing techniques, offering significantly higher performance (98% improvements, check out the blog [here](https://news.broadcom.com/mainframe-software/xcom-stop-transferring-data-like-its-1999)) compared to FTP. Also see [Broadcom's techdocs for XCOM](https://techdocs.broadcom.com/us/en/ca-mainframe-software/traditional-management/ca-xcom-data-transport-for-z-os/12-0.html).

XCOM excels with large batch files, in a reliable, auditable, managed way.


You can use items in this folder as XCOM examples for sending, receiving and running remote jobs. For example, you can replace the variables found in these members with actual values you need. 

The XCOM#INC, XCOM#JOB, XCOM#RCV, and XCOM#RUN members are constructed as "Models" for a Table Tool process. If you would like to see how you can use Table Tool to automate your XCOM processes, see recordings and links in the [Table Tool Examples](https://github.com/BroadcomMFD/broadcom-product-scripts/tree/main/endevor/Field-Developed-Programs/Table-Tool-Examples) folder.
These items support the [SonarQube-Interface-to-Endevor](https://github.com/BroadcomMFD/broadcom-product-scripts/tree/main/endevor/Field-Developed-Programs/SonarQube-interface-to-Endevor), also found on this GitHub site.
